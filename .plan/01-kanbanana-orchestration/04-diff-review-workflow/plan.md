# Phase 04 Plan: Diff Review Workflow

## Goal
Make ready-for-review the center of decision-making with a high-quality diff experience.

## Scope
1. Structured diff view in card detail.
2. File list and changed file navigation.
3. Review feedback input that returns task to in-progress.
4. Preserve review history across task cycles.

## Out of Scope
1. Commit and PR submission mechanics.
2. Dependencies and decomposition logic.

## Test Gate
1. Complete task and reach ready-for-review.
2. Inspect diff and submit feedback.
3. Verify task returns to in-progress with feedback context.
4. Confirm second completion returns to ready-for-review with updated diff.

## Exit Criteria
1. User can reliably review and iterate without leaving Kanbanana.
