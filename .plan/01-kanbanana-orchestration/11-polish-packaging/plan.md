# Phase 11 Plan: Polish and Packaging

## Goal
Prepare Kanbanana for broader use with onboarding clarity, packaging quality, and final UX polish.

## Scope
1. Philosophy onboarding card and first-run guidance.
2. Visual refinement and interaction polish across key flows.
3. Packaging quality for `npx kanbanana` distribution.
4. Developer and user docs for setup, workflow, and troubleshooting.
5. Minimal usage/subscription management page:
   - provider links
   - basic provider status surface
   - explicit note that advanced usage extraction is future R&D

## Out of Scope
1. Deep provider billing analytics.
2. Multi-agent race production rollout.

## Test Gate
1. Fresh machine run from `npx kanbanana` to first successful task completion.
2. Validate onboarding clarity with first-run checklist.
3. Validate minimal usage page links and provider status display.

## Exit Criteria
1. New user can understand and use core workflow without hand-holding.
2. Distribution experience is stable and repeatable.
